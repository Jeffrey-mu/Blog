# 【Linux环境上安装和升级Node】

# Linux环境安装Node

直接使用已编译好的包进行安装

Node 官网已经把 linux 下载版本更改为已编译好的版本了，我们可以直接下载解压后使用：

[node.js官网下载地址]

# cd /usr/local/src/
# wget https://nodejs.org/dist/v14.16.1/node-v14.16.1-linux-x64.tar.xz    // 下载
# tar xf node-v14.16.1-linux-x64.tar.xz         // 解压
# cd node-v14.16.1-linux-x64/                   // 进入解压目录
# ./bin/node -v                                 // 执行node命令 查看版本
v14.16.1

# ln -s /usr/local/src/node-v14.16.1-linux-x64/bin/node /usr/bin/node
# node -v

# ln -s /usr/local/src/node-v14.16.1-linux-x64/bin/npm /usr/bin/npm

# Linux环境升级Node

使用 n 模块来管理 node 的各种版本

查看当前 node 版本

$ node -v
v8.1.4 #初始node版本
1
2

# 1、通过 npm 安装 n：

$ npm install -g n
1

# 2、通过 n 安装指定版本:

$ n 8.11.2
1

# 3、再查看当前 node 版本：

$ node -v
v8.1.4 #和原来一样
1
2

# 升级异常

如果你跟我一样，发现 node 版本没有任何变化，那最有可能的情况就是，你的node的安装目录和 n 默认的路径不一样。

查看 node 当前安装路径：

$ which node
/opt/node/bin/node #举个例子
1
2

而 n 默认安装路径是 /usr/local，若你的 node 不是在此路径下，n 切换版本就不能把bin、lib、include、share 复制该路径中，所以我们必须通过 N_PREFIX变量来修改 n 的默认node安装路径。 编辑环境配置文件：

$ vim ~/.bash_profile
1

将下面两行代码插入到文件末尾：

export N_PREFIX=/opt/node #node实际安装位置
export PATH=$N_PREFIX/bin:$PATH
1
2

:wq 保存退出；

执行source使修改生效。

$ source ~/.bash_profile
1

确认一下环境变量是否生效：

$ echo $N_PREFIX
/opt/node
1
2

这时候我们需要重新安装：

$ n 8.11.2
1

再查看当前 node 版本

$ node -v
v8.11.2
1
2
